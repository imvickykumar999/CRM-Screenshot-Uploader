Here is a comprehensive step-by-step documentation for creating an .msi installer from an existing .exe file on Ubuntu using Wine and the WiX Toolset. This guide covers the entire process from start to end.

Creating an .msi Installer from .exe on Ubuntu

Prerequisites

  • Ubuntu (or any Linux distribution with apt package manager)
  • Wine installed for running Windows applications
  • WiX Toolset installer
  • An existing .exe file (in this example: screenshot_uploader.exe)

Steps:

1. Install Wine

Wine allows you to run Windows applications on Linux. Install it if you haven't already:

bash
sudo dpkg --add-architecture i386
sudo apt update
sudo apt install wine64 wine32

Verify the Wine installation:

bash
wine --version

2. Install Winbind

Winbind provides NTLM authentication support that some Windows applications may require. Install it with the following command:

bash
sudo apt install winbind

3. Download WiX Toolset

Download the WiX Toolset installer (wix314.exe) directly from the WiX GitHub page:

bash
wget https://github.com/wixtoolset/wix3/releases/download/wix3112rtm/wix314.exe

4. Install WiX Toolset Using Wine

Run the WiX Toolset installer using Wine:

bash
wine wix314.exe

Follow the installation prompts in the Wine environment.

5. Install Wine Mono

WiX Toolset may require .NET support, provided by Wine Mono. If not already installed, download and install Wine Mono manually:

bash
wget https://dl.winehq.org/wine/wine-mono/7.4.0/wine-mono-7.4.0-x86.msi
wine msiexec /i wine-mono-7.4.0-x86.msi

6. Prepare the WiX Source File (.wxs)

Create a directory for your project and move your .exe file into it. Then, create a .wxs file that will define how to build the .msi package.

Example directory setup:

bash
mkdir ~/wix_project
mv ~/Documents/screenshot_uploader/screenshot_uploader.exe ~/wix_project
cd ~/wix_project

Create the setup.wxs file:

bash
nano setup.wxs

Add the following content to setup.wxs:

xml
<?xml version="1.0"?>
<Wix xmlns="http://schemas.microsoft.com/wix/2006/wi">
  <Product Id="*" Name="ScreenshotUploader" Language="1033" Version="1.0.0.0" Manufacturer="YourCompany" UpgradeCode="YOUR-NEW-GUID-HERE">
    <Package InstallerVersion="200" Compressed="yes" InstallScope="perMachine" />

    <Media Id="1" Cabinet="media1.cab" EmbedCab="yes" />

    <Directory Id="TARGETDIR" Name="SourceDir">
      <Directory Id="ProgramFilesFolder">
        <Directory Id="INSTALLFOLDER" Name="ScreenshotUploader" />
      </Directory>
    </Directory>

    <Component Id="ProductComponent" Directory="INSTALLFOLDER" Guid="ANOTHER-NEW-GUID-HERE">
      <File Id="screenshot_uploader.exe" Source="screenshot_uploader.exe" KeyPath="yes" />
    </Component>

    <Feature Id="ProductFeature" Title="ScreenshotUploader" Level="1">
      <ComponentRef Id="ProductComponent" />
    </Feature>
  </Product>
</Wix>
  • Replace YOUR-NEW-GUID-HERE and ANOTHER-NEW-GUID-HERE with actual GUIDs, which can be generated using any online UUID generator.

7. Compile the .wxs File

Once your .wxs file is prepared, compile it into a .wixobj using the candle tool from WiX:

bash
wine "/home/bol7/.wine/drive_c/Program Files (x86)/WiX Toolset v3.14/bin/candle.exe" setup.wxs

If the command runs successfully, it will generate setup.wixobj.

8. Link the Compiled File to Create the .msi

Now, use the light tool to create the final .msi file from the .wixobj file:

bash
wine "/home/bol7/.wine/drive_c/Program Files (x86)/WiX Toolset v3.14/bin/light.exe" setup.wixobj

If you encounter validation errors, try suppressing them with the -sval option:

bash
wine "/home/bol7/.wine/drive_c/Program Files (x86)/WiX Toolset v3.14/bin/light.exe" -sval setup.wixobj

This should generate the setup.msi file in your project directory.

9. Test the Installer

Now that you have the .msi file, you can test it in Wine or on a Windows machine.

To test in Wine:

bash
wine msiexec /i setup.msi

To verify the installation, open the Wine File Explorer:

bash
wine explorer

Navigate to C:\Program Files (x86)\ScreenshotUploader and ensure that screenshot_uploader.exe has been installed there.

10. Transfer the .msi to a Windows Machine (Optional)

You can now transfer the .msi file to a Windows machine and run it there to ensure the installer works correctly on a native Windows environment.

Troubleshooting

  • NTLM Support Errors: Ensure that winbind is installed, which provides NTLM support.
  • File Path Issues: If you encounter issues related to file paths, ensure that the paths in the .wxs file are correct and not too long.
  • Use Wine Staging: If you encounter compatibility issues with Wine, consider using Wine Staging for better support of advanced features.

Conclusion

You have now successfully created an .msi installer for your screenshot_uploader.exe file using WiX Toolset on Ubuntu with Wine. If you encounter any errors or need further clarification, feel free to ask!
